2008.Jul.21, v1.6.3

New screeners can now be added to Screensaver via the web user interface!

A screener name search feature has been added to the menu panel.

Improvements to domain model for lab heads and labs, for better conceptual separation of lab members from labs/lab heads. Lab Head is now a first-class entity type. Assigning a lab to a screener or screen is now optional. Only lab heads can have lab members and lab affiliations, and only lab members can have lab heads.

New lab affiliations can be added via the web user interface (in Screen Viewer).

Improvements to the role mechanism, which now supports a hierarchy of "implied" roles. Added ScreensaverUser role, which is necessary for account login rights (allowing accounts to be enabled/disabled). Added LabHeadAdmin and NonScreener roles, and renamed some existing roles for clarity.

Most entity viewers now exclusively exist within the context of a search result, allowing the user to quickly search for and browse through other entities of the same type being viewed.

Wells can be flagged as 'deprecated', allowing Screensaver to warn (and potentially prevents) users from screening or cherry picking arbitrary sets of wells.

2008.Jun.17, v1.6.2

New screens can now be added to Screensaver via the web user interface (about time, eh?)

Screen Viewer enhancements for viewing, creating, and updating screens:

  • New publications can be added by specifying only a PubMed ID.
  • Attached Files can be added via browser file upload, or via cut & paste.
  • Additional Publishable Protocol fields added (in collapsible panel).
  • Billing Information can now be added, viewed, and edited (in collapsible panel).
  • Collaborators can be added and deleted similarly to other screen child entity types, with add and delete commands.
  • Lab Activities and Cherry Pick Requests data tables are limited to 10 rows, with commands added to view all in a separate search result page, allowing searching, filtering, and reporting to be performed on these screen child entity types.
  • Screens are now always viewed with the context of a screen search result, allowing user to scroll between screens at all times.
  • Screen editing is performed in a simplified view, outside of the search result data table, and with screen result panel, lab activities table, and cherry pick request table removed from view.

2008.May.20, v1.6.0

Activity Viewer has been added, allowing administrators to create, edit, and view Library Screening and RNAi Cherry Pick Screening activities. New activities can be created from the Screen Viewer page or the main menu. The Activities Browser now links each activity to the Activity Viewer page.

Volume fields now display and accept unit suffixes (e.g., uL and nL).

Date fields no longer store time of day (avoids time zone and date comparison issues).

Usability and cosmetic improvements to the data table navigation panel.

Cherry Pick Request "# unfulfilled lab cherry picks" value is now updated correctly if cherry picks are deleted.

Library Viewer's "Unload Library Contents" command has improved performance and reduced memory usage.

User activity log output improved, now includes date stamps, page names, and entity IDs. Improved coverage of activities.

New aqua color theme!

2008.Feb.11, v1.5.0

The implementation of Search Result data tables has been entirely re-designed, providing the following benefits:

  • Improved performance and scalability for large data sets. In particular, sorting and filtering of large data sets is performed by the database, not by Screensaver. Near minimal sets of database operations are performed in response to user actions (open, scroll, sort, add/remove columns, etc.).
  • Supports search results composed of columns across related entity types (e.g., Library, Well, Gene, Silencing Reagent), with user-configurable column selections, via a hierarchical column selection interface.
  • For developers, simplified definition of Search Result pages, with nearly-declarative implementation (simply provide a name and an entity property binding for each column). Dynamically configurable data access strategy ("load all" or "virtual scrolling/paging"). No data access code required (i.e., no Hibernate queries).

Users should note:

  • Screen Viewer's "Data" panel now has a full search interface, replacing the old "Show All, Show Plate, Show Positives" interface.
  • Screen Viewer's "Data" panel now supports combining data columns from multiple screen results (data headers) and studies (annotations).
  • Well Search Results pages should load faster for large data sets. For example, when viewing all wells for a library (accessed from the Library Viewer page).
  • Creative use of the Search and Column selection features provide a basic level of custom reporting throughout Screensaver.

2007.Dec.03, v1.4.0

Updated PubChem CIDs have been assigned to Screensaver's compounds, allowing the user to directly navigate to the latest PubChem entries for each compound. PubChem provides information on the biological activities of small molecules.

Search results can be filtered by specifying criterion for each column in the search result data table.

Screeners can now browse (and filter) all of their screening room activities in a search result page. Administrators can now browse (and filter) screening room users and staff in a search result page.

Administrator can now specify the set of rows to be left empty when Screensaver generates cherry pick plate mappings.

2007.Oct.11, v1.3.0

Upgraded the Screensaver object model to use Hibernate annotations instead of XDoclet annotations. Other various across-the-board model updates that should improve understanding, ease-of-use, and ease-of-maintenance of object model code for developers. This upgrade should have little to no impact on Screensaver users.

Users will be pleased that all data tables in Screensaver now present a consistent and improved navigation panel for interacting with the table. Developers will be pleased that all data tables are now implemented in terms of the same DataTable class.
